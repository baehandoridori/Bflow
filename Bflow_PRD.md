# Bflow - Product Requirements Document (PRD)

> **JBBJ 애니메이션 스튜디오를 위한 일정 관리 & 워크플로우 시각화 앱**

---

## 📌 Executive Summary

| 항목 | 내용 |
|------|------|
| **앱 이름** | Bflow |
| **타입** | PWA (Progressive Web App) |
| **대상** | Studio JBBJ 팀 (약 20명) |
| **핵심 가치** | 화려하고 명료하게 전체 상황을 확인하고, 누가 뭘 하는지 정확히 아는 것 |
| **브랜드 컬러** | `#F0E68C` (Khaki Gold) |
| **데이터 저장** | Google Drive 공유 드라이브 |

## 🎯 Problem Statement

### 현재 상황 (As-Is)
- 회의실 화이트보드 + 구두 메모로 일정 관리
- 줄글로 마일스톤/프로젝트 기간 기록 → 한눈에 파악 어려움
- 누가 무슨 작업을 하는지 추적 어려움

### 해결 목표 (To-Be)
- 시각적 타임라인으로 전체 일정 한눈에 파악
- 팀원별 작업 현황 실시간 확인
- 작업 간 의존성/연계를 노드 맵으로 시각화
- Slack 연동으로 리마인더 자동화

## 👥 Team Structure

### 팀 구성
- **인원**: 약 20명
- **역할**: 모두 애니메이터이면서 역할 분담
  - 릴 담당
  - 녹음 담당
  - 대본 담당
  - 애니메이팅 담당
  - 배경 담당
  - 에셋 준비 담당
  - 촬영 담당

### 작업 단위
- **대단위**: 에피소드 (예: "사코팍 ep.15")
- **소단위**: 씬(SC), 배경, 보정 등

| 단계 | 설명 | 예상 기간 |

| 애니메이션 | 본 애니메이션 작업 | 유동적 |

### View 상세

#### 1. Dashboard (메인)
- **오늘의 요약**: 마감 임박, 진행중, 완료 현황
- **간트 타임라인**: 에피소드별 진행률
- **미니 캘린더**: 이번 주/월 주요 일정
- **팀 현황**: 누가 뭘 하는지

#### 2. Timeline (간트 차트)
- 드래그 & 드롭으로 일정 조절
- 에피소드 > 파이프라인 단계 > 태스크 계층
- 줌 인/아웃 (일/주/월 단위)
- 마일스톤 마커

#### 3. Calendar
- 월간/주간 뷰 전환
- 마감일, 마일스톤, 회의 표시
- 휴가/출장 등 부재 일정

#### 4. Team View
- 팀원 리스트 + 현재 작업
- 프로필 사진/아바타
- 워크로드 시각화

#### 5. Node Map (보조 뷰)
- **옵시디언 그래프 뷰 스타일**
- 작업 간 의존성/연계 시각화
- 드래그로 노드 배치
- 줌/팬 인터랙션
- 커스텀 연결선 생성

## 📋 Task Structure

### 태스크 예시
```
- XXX 편 SC_001 - 홍길동
- XXX 편 배경 제작 - 주인공 집 - 김길동
- XXX 편 보정 - 배길동
- XXX 편 릴 제작 - 이길동
```

### 태스크 필드

| 필드 | 필수 | 설명 |
|------|------|------|
| 제목 | ✅ | 태스크 이름 |
| 담당자 | ✅ | 팀원 지정 |
| 마감일 | ✅ | 데드라인 |
| 진행 상태 | ✅ | 커스텀 가능 (기본: 대기/진행중/완료) |
| 우선순위 | ❌ | 선택적 |
| 메모/코멘트 | ❌ | 자유 입력 |
| 연결된 태스크 | ❌ | 노드 맵용 의존성 |

### 진행 상태 (커스터마이징 가능)
- 대기
- 진행중
- 리뷰/피드백
- 완료

## 🎨 Design Specifications

### 테마 시스템
- **다크 모드** / **라이트 모드** 전환
- **커스텀 액센트 컬러** 변경 가능
- 기본 브랜드 컬러: `#F0E68C`

### 디자인 방향
- **베이스**: Slack 기본 앱 느낌
- **UX 참고**: Monday.com, Notion
- **아이콘**: SVG 커스텀 (이모지 사용 안 함)
- **톤**: 프로페셔널하면서 친근함

### 인터랙션 & 애니메이션

| 요소 | 효과 |
|------|------|
| 페이지 전환 | 부드러운 슬라이드/페이드 모션 |
| 드래그 앤 드롭 | 물리 기반 바운스 효과 |
| 마일스톤 달성 | 파티클/컨페티 이펙트 |
| 진행률 바 | "흐르는" 애니메이션 |
| 카드 호버 | 3D 틸트 + 살짝 떠오름 |
| 로딩 | 스켈레톤 UI + 스무스 페이드인 |
| 노드 맵 | 물리 시뮬레이션 (옵시디언 스타일) |

### 영감 레퍼런스
- Raycast 홈페이지 - 마우스 따라오는 인터랙션
- Obsidian Graph View - 노드 물리 시뮬레이션
- Linear - 깔끔한 간트 차트
- Slack - 전체적인 UI 톤

## ⚠️ Constraints & Non-Goals

### 절대 안 됨 (Anti-patterns)
- ❌ 바이브코딩 느낌의 이모지 범벅 UI
- ❌ 복잡하고 뭐가 뭔지 모르는 UI
- ❌ 버벅거리거나 끊기는 애니메이션
- ❌ 학습이 필요한 복잡한 인터페이스

### 범위 외 (Out of Scope)
- 외부 협업자 접근
- 다른 채널(장삐쭈) 관리
- 복잡한 권한 시스템
- 오프라인 완전 지원

## 🛠️ Technical Stack

### Frontend
- **Framework**: React 18 + TypeScript
- **Styling**: Tailwind CSS
- **Animation**: Framer Motion
- **State**: Zustand 또는 Jotai
- **Charts**: D3.js (간트, 노드맵)

### PWA
- Service Worker for caching
- Web Push Notifications
- Installable (manifest.json)

### Data Layer
- Google Drive API (파일 읽기/쓰기)
- 또는 로컬 파일 시스템 접근 (File System Access API)

### Integrations
- Slack Incoming Webhooks

1. **명료성**: 5초 내에 전체 프로젝트 상황 파악 가능
